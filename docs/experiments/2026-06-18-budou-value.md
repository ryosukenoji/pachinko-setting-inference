# 実験ノート: ぶどうカウントの価値（クロールデータで設定判別できるか）

Date: 2026-06-18
Script: [`scripts/exp_budou_value.py`](../../scripts/exp_budou_value.py)
Seed: 0 / 試行: 400（4v5は200, top1は400/6設定）

## 動機

クロール可能な実戦データサイト（rx7038型: 台番号ごとの総スタート/BB/RB、shimakazu型: 機種別月間集計）は
**BIG/REG/総回転は出すが「ぶどう回数」が無い**ことを調査で確認（[design_proposal §4.1](../design_proposal.md)）。
ジャグラーの設定看破はぶどう確率の設定差を主信号にするとされるため、
**「ぶどう抜き（クロール相当）でどれだけ推定が劣化するか」を機種別に数値化**し、
クロールデータだけで「打つ/やめる」判断ができるかを検証した。

## 方法

- 真の設定からゲーム結果を multinomial サンプリング（ぶどうも含む完全な台）。
- 推定側で観測量を 2 条件に切り替え（`compute_posterior(..., events=...)`）:
  - **ぶどう込み**: {BIG, REG, ぶどう}（手入力相当）
  - **ぶどう抜き**: {BIG, REG}（クロール相当。ぶどうは "other" に吸収）
- 指標:
  1. 高設定(4-6) vs 低設定(1-3) の2択判別精度（= 打つ/やめる に直結）
  2. それを90%にする必要回転数
  3. 全6設定 top-1 的中率（N=8000）
  4. 参考: 設定4 vs 5 の厳密分離（N=40,000）
- 対象機種: マイジャグラーV / アイムジャグラーEX（いずれも尤度テーブルは `[unverified]`）

> **改訂 (2026-06-18, 同日)**: 初回は `[unverified]` の誤った尤度テーブル（マイジャグラーVのREG列が1設定ずれ・機械割過小・ぶどうがIII相当）で実施し、「マイジャグラーはぶどう+6〜8pt価値」という結論を得た。その後 jugjug.net + juggler7.com の2ソースでテーブルを修正（[my_juggler_v.json](../../data/machines/my_juggler_v.json) provenance 参照）して再実行したところ、**マイジャグラーVの結論が覆った**。以下は修正後の値。**この覆り自体が裏取り（検証規律）の価値の実証**。

## 結論（修正後）

1. **「打つ/やめる」（高/低判別）に限れば、両機種ともクロールの BIG/REG だけで実用域**。ぶどう手入力との差は **0〜3pt** に収まる。
2. **高/低判別ではぶどうの限界価値は両機種とも小さい（差0〜3pt）**。ただし用途で割れる:
   - **マイジャグラーV**: 修正後のREG設定差が大きく（1/409.6→1/229.1, 約1.8倍）、主信号がREGに乗る。高/低も厳密IDもぶどう抜きでほぼ同等。高/低90%必要N 約25,000G。
   - **アイムEX**: REG（1/439.8→1/255.0）が主信号。**ぶどうは設定1-5がほぼ同値（1/6.01）で設定6のみ優遇（1/5.80）** → ぶどうは「設定6フラグ」としてのみ機能。高/低判別には +0.5〜1.5pt しか効かないが、**厳密 top-1 ID には +8.4pt 効く（設定6を当てるため）**。
   - → **高/低（打つ/やめる）はどちらもクロールのBIG/REGでほぼ代替可能。ぶどう手入力が効くのは「設定6を厳密に当てたい」時だけ**。
3. **本当の制約はぶどう欠落ではなく「サンプル量と鮮度」**: 高/低90%判別に **2.5〜4万回転**必要。1日数千ゲームでは70〜80%止まり。厳密な設定ID（top-1）はぶどう込みでも40%台。
   - 補足: アイムEXの 4 vs 5 厳密分離が 89.5% と高いのは、修正後REGが設定4(1/315.1) vs 設定5(1/255.0) で大きく開くため（旧誤テーブルでは 1/331→1/315 とほぼ同値で 68% だった＝ここでもテーブル誤差が結論を変えた）。
4. **最大の教訓 = 裏取りの価値**: 初回の「マイジャグラーはぶどう+6〜8pt価値」という結論は、**誤った尤度テーブルの産物**だった。REG列が1設定ずれていたため REG信号が過小評価され、相対的にぶどうの価値が水増しされていた。正しいテーブルでその結論は消滅。**テーブル誤差が実験結論を反転させ得る**ことの実証（[design_proposal §1 の「未知だが固定された隠れパラメータ」推定で、固定パラメータ＝尤度テーブルの正確性が全ての土台](../design_proposal.md)）。

## 注意・限界

- 数値は尤度テーブル（攻略サイト解析値）に依存。両機種とも2ソース照合済みだが、**ぶどうは両機種とも公式非公表の実測推定**（サイト間でばらつく。アイムEXは設定6値が 1/5.80〜5.83 で割れる）。
- multinomial 独立サンプリングの理想化。実機のヒキの偏り・打ち手の取りこぼし（ぶどう手入力のカウントミス）は未考慮。ぶどう抜きの方がカウントミスが無い分、現実では差がさらに縮む可能性。
- seed=0 単一。本格運用前に複数seedで分散を確認すべき。
