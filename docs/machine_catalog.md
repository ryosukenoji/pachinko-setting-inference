# 機種カタログ（横展開リスト）

設定看破の横展開対象。**「公式参照(BIG/REG)で主信号が足りるか」** と **版(世代)** で整理する。
エンジンは機種非依存なので、追加は `data/machines/<id>.json` を1枚足すだけ（コード変更不要）。
ただし分散モデル(`session_pnl`/`daily`)は **A-normal タイプ専用**（BIG/REG・3枚掛け前提）。

## 信号と公式参照度の早見

- **REG主信号 + 公式で足りる** → 確度高・最優先（ジャグラー系がここ）
- **小役主信号** → 公式に小役が非公表のことが多く、実測推定が必須 → 確度中
- **AT/ART(状態依存)** → ベイズ前提が崩れる → 対象外（別設計）

## 対象外（状態依存=ベイズ前提が崩れる）

- 6号機AT/ART機（番長ZERO、北斗、バジリスク絆2 等）
- スマスロ全般（有利区間・モード・天井 + 高純増で分散モデルも別物）
- → やるなら「状態依存の尤度」「AT純増ベースの分散モデル」を別途設計（[experiments の世代差メモ参照]）

## 追加手順（チェックリスト）

1. **版を確定**（V/IV、5号機/6号機 を取り違えない。`generation` 必須）
2. 公式 or 攻略2サイトで **BIG/REG/合算/機械割** を取得し、合算=BIG+REG で内部整合を確認
3. **ぶどう**は攻略サイト実測を2ソース相互照合（割れたら provenance に記録）
4. `payout_coins`（ジャグラー系は BIG=240/REG=96/3枚）
5. `provenance.status` を `cross-checked` にし `sources` を列挙
6. `python3 -m unittest tests.test_tables`（全テーブル整合チェック）が通ること
